# 시스템 리소스 및 로그 분석기 (System Resource & Log Analyzer)

## 개요 (Overview)
이 프로젝트는 Python과 Pandas를 활용하여 구축한 엔드투엔드(End-to-End) 데이터 엔지니어링 및 분석 파이프라인입니다. 시스템 리소스 로그를 생성하고, 가공되지 않은 문자열 데이터를 정제하며, 시계열 분석을 수행하여 지속적인 하드웨어 부하 및 잠재적인 시스템 장애를 감지함으로써 실제 서버 환경을 시뮬레이션합니다.

이 도구는 로그 파싱, 데이터 조작 및 자동화된 시스템 상태 보고에 대한 실무 기술을 보여줍니다.

## 주요 기능 (Key Features)
* **맞춤형 데이터 생성:** 타임스탬프, 서버 ID, 리소스 지표가 포함된 10,000개의 무작위 더미 서버 로그 데이터를 생성하는 독립적인 Python 스크립트(`LogAnalysis.py`).
* **자동화된 데이터 정제:** 포맷이 통일되지 않은 원본 로그 문자열(예: `" 85% "`, `" 2048 MB "`)을 분석 가능한 구조화된 숫자 데이터 타입으로 변환.
* **시계열 분석:** Pandas의 `datetime` 인덱싱을 활용하여 복잡한 시간순 연산 수행.
* **지속적 부하 감지:** 슬라이딩 윈도우 함수(이동 평균)를 구현하여 일시적인 CPU 사용량 급증과 치명적이고 지속적인 리소스 고갈 상태를 명확히 구분.
* **자동화된 분류 및 보고:** 정상 상태의 로그를 필터링하고 치명적인 경고만 서버별로 분리된 CSV 보고서로 자동 내보내기.
* **데이터 시각화:** Matplotlib을 사용하여 추세선을 생성하고 시스템 부하 임계값에 도달하기까지의 정확한 이벤트를 시각화.

## 기술 스택 (Tech Stack)
* **언어:** Python
* **라이브러리:** Pandas (데이터 조작), Matplotlib (데이터 시각화)
* **환경:** Jupyter Notebook

## 프로젝트 구조 (Project Structure)
* `LogAnalysis.py`: 데이터 생성 스크립트. 원본 데이터셋을 생성하기 위해 가장 먼저 실행합니다.
* `LogAnalyzer.ipynb`: 데이터 파이프라인, 필터링 로직 및 시각화 코드가 포함된 핵심 분석 노트북.
* `Server_Log.csv`: Python 스크립트에 의해 생성된 원본 출력 파일 (일반적으로 Git 업로드 시 `.gitignore`에 추가하여 제외).
* `Server_A_High_Usage.csv` / `Server_B_High_Usage.csv` / `Server_C_High_Usage.csv`: 치명적인 시스템 경고만 포함된 정제 및 자동 생성된 내보내기 파일.
* `Server_A_Crash_Report.png`: 시뮬레이션된 CPU 장애로 이어지는 10-tick 이동 평균 트렌드를 시각화한 자동 생성 이미지.

## 실행 방법 (How to Run)
1. 이 저장소(repository)를 로컬 환경에 클론합니다.
2. 분석에 필요한 패키지가 설치되어 있는지 확인합니다: `pip install pandas matplotlib`
3. 터미널에서 `python LogAnalysis.py`를 실행하여 새로운 가상 로그 데이터를 생성합니다.
4. 선호하는 Jupyter 환경(예: VS Code)에서 `LogAnalyzer.ipynb`를 열고 셀을 순차적으로 실행하여 데이터를 정제하고, 이동 평균을 계산하며, 시각화 자료를 생성합니다.
